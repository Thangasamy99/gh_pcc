BACKEND CHANGES
===============================================================================

1. NEW MODELS CREATED
-------------------
- /backend/src/main/java/com/hospital/security/model/Entry.java
  * Entity for tracking all entries (NORMAL, EMERGENCY, VISITOR)
  * Fields: id, entryId, entryType, fullName, gender, age, phoneNumber, address, etc.
  * Relationships: branch, registeredBy

- /backend/src/main/java/com/hospital/security/model/Visitor.java
  * Entity for visitor management
  * Fields: id, visitorId, visitorName, patientName, relationship, etc.

- /backend/src/main/java/com/hospital/security/model/VisitorStatus.java
  * Enum: ACTIVE, COMPLETED, CANCELLED

2. NEW DTOs CREATED
------------------
- /backend/src/main/java/com/hospital/security/dto/DynamicEntryRequestDTO.java
  * Request DTO for creating entries
  * Validation annotations for required fields

- /backend/src/main/java/com/hospital/security/dto/EntryResponseDTO.java
  * Response DTO for entry data

- /backend/src/main/java/com/hospital/security/dto/VisitorResponseDTO.java
  * Response DTO for visitor data

- /backend/src/main/java/com/hospital/security/dto/SecurityDashboardDTO.java
  * Dashboard statistics DTO

- /backend/src/main/java/com/hospital/security/dto/PatientLocationDTO.java
  * Patient location tracking DTO

- /backend/src/main/java/com/hospital/security/dto/MetadataDTO.java
  * Metadata for dropdowns (added @Builder annotation)

3. NEW REPOSITORIES
------------------
- /backend/src/main/java/com/hospital/security/repository/EntryRepository.java
  * Methods: findByBranchId(), findByBranchIdAndEntryTimeBetween(), etc.

- /backend/src/main/java/com/hospital/security/repository/VisitorRepository.java
  * Visitor-specific query methods

4. NEW SERVICES
--------------
- /backend/src/main/java/com/hospital/security/service/SecurityService.java
  * Interface defining all security operations

- /backend/src/main/java/com/hospital/security/service/impl/SecurityServiceImpl.java
  * Complete implementation with business logic
  * Methods: createEntry(), getDashboardStats(), getAllPatientEntries(), etc.

5. NEW CONTROLLERS
-----------------
- /backend/src/main/java/com/hospital/security/controller/SecurityController.java
  * REST endpoints for entry management
  * Endpoints: POST /entries, GET /entries/today/{branchId}, GET /entries/patients/{branchId}

- /backend/src/main/java/com/hospital/security/controller/SecurityMetadataController.java
  * Metadata endpoints for dropdowns

API ENDPOINTS
===============================================================================

AUTHENTICATION:
- POST /api/v1/auth/login - User login
- GET /api/v1/auth/session/{id}/validate - Session validation

SECURITY OPERATIONS:
- POST /api/v1/security/entries - Create new entry
- GET /api/v1/security/entries/today/{branchId} - Get today's entries
- GET /api/v1/security/entries/patients/{branchId} - Get all patient entries
- GET /api/v1/security/entries/{id} - Get entry by ID
- GET /api/v1/security/dashboard/{branchId} - Get dashboard statistics
- GET /api/v1/security/visitors/active/{branchId} - Get active visitors
- POST /api/v1/security/visitors/{id}/checkout - Checkout visitor

METADATA (Public):
- GET /api/v1/security/metadata/entry-types - Entry type options
- GET /api/v1/security/metadata/visit-types - Visit type options
- GET /api/v1/security/metadata/departments - Department options
- GET /api/v1/security/metadata/emergency-types - Emergency type options
- GET /api/v1/security/metadata/genders - Gender options
- GET /api/v1/security/metadata/relationships - Relationship options
